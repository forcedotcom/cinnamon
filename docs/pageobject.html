PageObject

global virtual with sharing class PageObject
Description PageObject is an abstract base class that is used to define a PageObject class for a given UI page. All PageObject classes in Cinnamon must inherit from this class.

To instantiate a PageObject class, Context.getPageObject method must be used with the type of a PageObject class being passed in to the method as an argument

   getPathAndQuery

global virtual String getPathAndQuery(Map< String , String > params)
Description Return the initial url path and query part for the Page Object. This method is often used to implement BaseTest.getStartingPath to indicate what is the first url to be loaded by the browser
Parameters
params is a Map that contains values that maybe used during the construction of the url. For example: public with sharing class TeamDetailPageObject extends cinnamon.PageObject{ public override String getPathAndQuery(Map params){ PageReference page = new ApexPages.StandardController(params.get('team')).edit(); return page.getUrl(); } } public abstract with sharing class ScrumTeamBaseTest extends cinnamon.BaseTest{ public static String TEAM_NAME = 'Cinnamon'; public override void setup(cinnamon.Context context) { String teamId = MyTestUtil.createTeam(TEAM_NAME); context.put('teamId', teamId); } public override String getStartingPath(cinnamon.Context context) { Map params = new Map (); params.put('team', context.get('teamId')); return getPageObject(context).getPathAndQuery(params); } }
