VisualforceLocator

global with sharing class VisualforceLocator extends ElementLocator
Description VisualforceLocator is used to locate an HTML element generated by a Visualforce tag. It must be passed in as an argument to getElement method in PageObject class to find and return a WebElement object that represents a certain HTML element.

The following Visualforce tags are currently supported by VisualforceLocator:

  • apex:selectList
  • apex:outputField
  • apex:pageBlockTable
  • apex:commandButton
  • apex:pageMessages
  • apex:inputField
For other Visualforce tags not listed above, VisualforceByIdLocator must be used to locate an element, instead.
See WebElement , VisualforceByIdLocator

For example, the following code snippet shows how to retrieve a WebElement that represents an input element generated by <apex:inputField> tag with 'name' set to its Id attribute 

cinnamon.WebElement nameField = getElement(new VisualforceLocator('apex:inputField', 'name');

   VisualforceLocator

global VisualforceLocator ( String vfTag, String vfId, String attributeExpression)
Parameters
vfTag The Visualforce tag used to generate a given HTML element
vfId Id attribute of the Visualforce tag being used
attributeExpression An expression used to locate an element based on their attributes and the values of those attributes.

For example, the following code snippet shows how to retrieve a Select Option element whose value is "USA", 

cinnamon.WebElement optionUSA = getElement(new VisualforceLocator('apex:selectOptions', null, 'value="USA"'));
